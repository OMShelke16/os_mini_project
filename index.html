What is Page Replacement?

When there’s not enough space in physical memory (RAM) for all the program pages, the operating system needs to decide which pages to remove to bring in new ones. This is called page replacement.

Goal: Try to reduce the number of times you have to load a page from disk (page faults).

Key Terms

TermMeaning
PageA block of program data that can be loaded in RAM
FrameA slot in RAM where a page can sit
Page FaultThe needed page is not in RAM, so it must be loaded
Page HitThe needed page is already in RAM
Hit RatioNumber of hits compared to total requests
ThrashingToo many page faults, making the computer very slow

FIFO — First In, First Out

Simple idea: Remove the page that’s been in memory the longest.
How it works: Imagine a line of pages. When a new page needs space, kick out the one at the front of the line and add the new one to the end.

  • Easy to use
  • Doesn’t care how often a page is needed
  • Can sometimes make things worse by adding more memory (Bélády's Anomaly)

LRU — Least Recently Used

Simple idea: Remove the page that hasn’t been used in the longest time.
How it works: Keep track of when you last used each page. If you need space, remove the page you used the longest time ago.

  • Works well in practice
  • No Bélády's Anomaly
  • Needs some way to track usage times

Optimal (Bélády's Algorithm)

Simple idea: Remove the page that will not be used for the longest time in the future.
How it works: Look ahead in the reference string, and pick the page that is needed farthest in the future (or not at all).

  • Best possible method
  • Impossible in real life because we can’t see the future
  • Great for comparing other algorithms
